# WatchRSS 日志上传说明

更新日期：2026年3月13日  
生效日期：2026年3月13日

欢迎使用 WatchRSS（以下简称“本应用”）。本说明用于向您说明本应用日志上传功能的触发方式、处理范围、使用目的和您的相关选择权。本说明仅适用于您主动使用日志上传功能的场景，不适用于本应用的一般浏览、阅读、登录或订阅功能。

如果您不同意本说明，您可以不进入日志上传页面，或进入后不点击“开始上传”，这不会影响您继续使用本应用的其他基本功能。

## 一、日志上传功能的触发方式

1. 本应用不会默认在后台自动上传日志。
2. 日志默认仅保存在您的设备本地。
3. 仅当您主动进入“隐私及防诈说明”页面，并在知悉相关提示后点击“开始上传”按钮时，本应用才会处理并上传日志。
4. 点击“开始上传”后，应用会跳转至上传页并自动开始上传流程。
5. 本功能仅建议在开发者明确要求您协助排查问题时使用。

## 二、我们可能处理的日志信息

在您主动触发上传后，本应用可能处理以下与故障排查、兼容性分析和性能优化相关的信息：

1. 应用运行日志、错误日志、异常堆栈。
2. 与功能执行过程相关的状态信息、时间信息、操作结果信息。
3. 为定位问题所必需的设备和运行环境信息，例如系统版本、应用版本、页面状态、功能调用结果等。
4. 与日志检索相关的必要元数据，例如取件码、创建时间、字符数、文件大小等。

请注意：

1. 日志处理原则上不以收集您的身份信息为目的。
2. 但日志内容可能在特定故障场景下包含您主动操作后产生的业务上下文信息，因此您仍应仅在必要时上传。
3. 请勿在与开发者沟通时额外主动提供与排障无关的敏感信息。

## 三、日志处理与传输方式

根据当前版本实现，您主动上传日志时，日志会按如下方式处理：

1. 从设备本地读取待上传日志内容。
2. 对日志内容进行压缩处理。
3. 使用对称加密方式对日志内容进行加密。
4. 使用内置公钥对加密密钥进行再次加密。
5. 上传加密后的日志文件，并记录日志检索所必需的元数据。

根据当前代码资源可见的实现痕迹，日志上传页面存在 **InstantDB** 相关接口与存储结构，用于承载加密后的日志文件和索引信息。就对外披露口径而言，相关处理规则以本说明和《隐私政策》为准。

## 四、日志上传的目的

我们仅会将您主动上传的日志用于以下目的：

1. 排查应用崩溃、卡顿、登录异常、同步失败、兼容性问题等故障。
2. 分析影响稳定性与性能的问题成因。
3. 改进产品质量、修复程序错误和提升使用体验。

未经您的明确授权，我们不会将您主动上传的日志用于广告营销、个性化推荐或与故障排查无关的其他用途。

## 五、存储地域、跨境传输与保存期限

1. 您主动上传后的加密日志文件及相关索引信息存储于位于**俄勒冈州，美国**的服务器。
2. 前述处理涉及**跨境传输**。
3. 已上传日志原则上自上传完成起保存 **24 小时**，届满后删除或作不可逆处理。
4. 设备本地日志的保存期限以本应用本地日志机制实际留存情况为准。

2. 为核验请求，我们可能要求您提供取件码、上传时间、设备型号、问题描述等必要信息。
3. 我们将在法律法规允许和技术可行的范围内尽快处理，通常不超过 **15 个工作日**。

## 七、您的选择权

关于日志上传功能，您享有以下选择权：

1. 您可以选择不使用日志上传功能。
2. 您可以在进入说明页后不点击“开始上传”。
3. 您可以通过前述渠道申请查询、删除或停止进一步处理已上传的日志信息，但法律法规另有规定的除外。

您拒绝使用日志上传功能，不会影响 RSS 基础阅读等核心功能的正常使用。
