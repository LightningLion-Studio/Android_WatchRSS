## 3. 需要在文档中单独披露的功能点

### 3.1 账号与登录

- 本应用当前无自有账号体系。
- 用户可主动登录 B 站账号以使用个性化内容、点赞、投币、收藏、稍后再看等能力。
- 用户可主动登录抖音以使用相关内容浏览能力。
- B 站与抖音登录态相关 Cookie 属于敏感网络身份信息，应在隐私政策中明确说明存储目的、方式、使用场景和删除方式。

### 3.2 本地存储与同步

- RSS 订阅源、阅读进度、收藏、稍后再看、阅读设置会保存在本地设备。
- 收藏/稍后再看和“从手机输入”存在局域网同步能力，当前由手表本地 HTTP 服务提供，不经过公司自建云服务器。
- 该能力应写明为“用户主动触发的设备间局域网传输”。

### 3.3 日志上传

- 日志默认仅保存在用户设备本地。
- 仅当用户主动进入“隐私及防诈说明”页面并点击“开始上传”时，日志才会被处理和上传。
- 当前前端资源显示日志会先压缩、再进行对称加密，并使用内置公钥加密密钥后上传。
- 当前代码痕迹显示日志上传依赖第三方服务 InstantDB。
- 正式隐私政策和日志上传说明中需要单列：
  - 上传触发条件
  - 上传内容范围
  - 上传目的
  - 第三方服务商
  - 用户撤回或删除路径

### 3.4 分享与外部跳转

- 应用支持系统分享或二维码分享。
- RSS、B 站、抖音内容可能包含原始链接、分享链接或浏览器跳转。
- 协议中需要说明第三方内容和第三方网页由对应提供方负责。

## 4. 文案口径建议

- 用户协议：
  - 侧重服务边界、账号与行为规范、第三方内容责任、服务变更、免责、知识产权、争议解决。
- 隐私政策：
  - 侧重个人信息类型、处理目的、使用场景、存储期限、共享/传输、用户权利、未成年人保护。
- 日志上传说明：
  - 作为独立说明或隐私政策附录，单独强调“自愿、临时、仅限排障、非默认后台上传”。
